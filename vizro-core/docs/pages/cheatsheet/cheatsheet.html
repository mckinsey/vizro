Vizro - Cheatsheet

Essential commands & syntax reference

Basic dashboard

Installation
Install the Vizro package to get started with dashboard creation.
pip install vizro Install Vizro
Simple dashboard
This code creates a complete dashboard with pages & components, and runs the application. All other code in this cheatsheet are snippets to illustrate the feature under discussion. 
from vizro import Vizro
import vizro.models as vm

dashboard = vm.Dashboard(
    pages=[
        vm.Page(
            title="My first page!",
            components=[vm.Card(text="Welcome to Vizro!")],
        )
    ]
)

Vizro().build(dashboard).run()

Controls

Filters
Apply data filtering to components based on column values using various selector types. 
import vizro.models as vm

filter = vm.Filter(
    column="species",
    selector=vm.Dropdown(
        value=["setosa", "versicolor"]
    ),
)
Filter tips:
• column argument denotes the target column to be filtered
• Set targets to specify which components on the page the filter should apply to.
Parameters
Dynamically control component properties through user input selectors. 
import vizro.models as vm

parameter = vm.Parameter(
    targets=["scatter_chart.title"],
    selector=vm.Dropdown(
       options=["My scatter chart", "A better title!", "Another title..."],
       multi=False,
    ),
)
Parameter tips:
• Specify targets in the form target_component_id.target_argument (for example, scatter_chart.title)
• To make parameter optional you can specify the string "NONE" in the options or value field
• To change nested parameters, you can specify the targets argument with a dot separated string like target_component_id.target_argument.first_hierarchy
• selector argument is a mandatory field, specify it by providing the appropriate model and the desired options/numeric ranges.
Categorical selectors
Interactive components for selecting from discrete categories and options. 
selector=vm.Dropdown()
selector=vm.Checklist()
selector=vm.RadioItems()
Numerical selectors
Components for selecting numerical values and ranges with precise control. 
selector=vm.Slider()

Temporal selectors
Date and time selection components for temporal data filtering and analysis. 
selector=vm.DatePicker()
selector=vm.DatePicker(range=False)

Page layout & navigation

Multi-page dashboard
Create dashboards with multiple pages and automatic navigation between them. 
page_1 = vm.Page(
    title="Overview",
    components=[
        vm.Card(text="Test")
    ]
)

page_2 = vm.Page(
    title="Details",
    components=[
        vm.Card(text="Test")
    ]
)

page_3 = vm.Page(
    title="Summary",
    components=[
        vm.Text(text="Test")
    ]
)

dashboard = vm.Dashboard(title="My dashboard",pages=[page_1, page_2, page_3])
Navigation
Configure custom navigation to group pages together and customize how they appear in your navigation by specifying a navigation model. 
navigation = vm.Navigation(
    pages=[page_1, page_2, page_3],
    nav_selector=vm.NavBar(
        items=[
            vm.NavLink(
                icon="home",
                label="Home",
                pages=["Overview"],
            ),
            vm.NavLink(
                icon="Library Add",
                label="Features",
                pages={
                    "Components": ["Text"],
                    "Controls": ["Details"],
                },
            )
        ]
    )
)
Navigation tips:
• To include only some of your dashboard pages in your navigation then list them in the pages argument of the vm.Navigation model.
• Alter the icons used by specifying the name of the icon in the Google Material icons library.
• Group your pages together by specifying them as a dictionary.
Grid layout
Arrange components in grid patterns with precise positioning and responsive design with grid layout. 
grid_layout = vm.Grid(
    grid=[
        # All rows must be of same length
        [0, 1, -1],
        [2, 2, 2],
        [2, 2, 2],
        [2, 2, 2],
    ],
)
grid = [[0], [1], [2], [3]
grid = [[0, 1, 2, 3]]
Layout tips:
• Grid numbers match component positions
• Add empty spaces within grid layout by specifying -1 e.g grid=[[0], [1], [-1]]
• Control the scroll behavior by specifying row_min_height and col_min_width
Flex layout
Achieve flexible layout system for dynamic component arrangement and responsive behavior with flex layout. 
flex_layout = vm.Flex(
    direction="row"
)
Flex layout tips:
• To wrap components into multiple lines use wrap=True
• Control the spacing between components in the flex container by using gap argument

Data management

Data loading
Load and manage data from various sources using the centralized data manager. 
import pandas as pd
from vizro.managers import data_manager

# Load data
df = pd.read_csv("data.csv")
data_manager["my_data"] = df

# Use in components
vm.Graph(
    figure=px.scatter(
        data_frame="my_data",
        x="x_col",
        y="y_col"
    )
)
Dynamic data
A dynamic data source is a Python function that returns a pandas DataFrame and can be re-executed to refresh dashboard data without restarting it. 
import pandas as pd
from vizro.managers import data_manager

# Load data
def load_iris_data():
    iris = pd.read_csv("iris.csv")
    return iris.sample(50)

data_manager["iris"] = load_iris_data

# Use in components
vm.Graph(
    figure=px.box(
        "iris",
        x="species",
        y="petal_width",
        color="species",
    )
)
Data integration:
• You can give arguments to your dynamic data loading function that can be modified from the dashboard.
• SQL databases via pandas
• Kedro Data Catalog integration

Actions

Vizro actions are similar to callbacks in Dash.
Export data
To enable downloading data, you can add the export_data action to the Button component. 
# Export data action
from vizro.actions import export_data

actions=export_data()
Cross-filtering
Cross-filtering enables you to click on data in one chart or table to filter other components in the dashboard. 
# Cross-filtering
from vizro.actions import filter_interaction

actions=filter_interaction(targets=["your-target-component-id"])
Actions tips:
• Chain multiple actions together by providing them in sequence you want them to be executed. Next action in order will execute only after the previous action has completed.

Custom Components

Custom charts
Create custom plotly charts with advanced styling and interactive features beyond standard templates. 
from vizro.models.types import capture
import vizro.plotly.express as px

df = px.data.iris()

@capture("graph")
def scatter_with_line(data_frame, x, y, color=None, size=None, hline=None):
    fig = px.scatter(data_frame=data_frame, x=x, y=y, color=color, size=size)
    fig.add_hline(y=hline, line_color="gray")
    return fig

# Use in Graph component
vm.Graph(figure=custom_chart(data_frame=df))
Custom Components
Extend any Vizro component or create completely new ones using any dash-compatible component. 
from typing import Literal
from dash import html
import vizro.models as vm

class Jumbotron(vm.VizroBaseModel):
    type: Literal["jumbotron"] = "jumbotron"
    title: str
    subtitle: str
    text: str

    def build(self):
        return html.Div(
            [
                html.H2(self.title),
                html.H3(self.subtitle),
                html.P(self.text),
            ]
        )

# Register the component
vm.Page.add_type("components", Jumbotron)
Custom Actions
Implement custom callback functions that respond to user interactions and update dashboard state. 
from vizro.models.types import capture
import vizro.models as vm

@capture("action")
def update_card_text(species):
    return f"You selected species **{species}**"

# Use with Action
vm.Action(
    function=update_card_text("my_selector"),
    outputs="my_card",
)
Custom table
Build custom data tables with specific formatting, styling, and interactive features. 
from dash_ag_grid import AgGrid
from vizro.models.types import capture
import vizro.models as vm

df = px.data.gapminder()

@capture("ag_grid")
def my_custom_aggrid(chosen_columns: list[str], data_frame=None):
    defaults = {
        "className": "ag-theme-quartz-dark ag-theme-vizro",
        "defaultColDef": {
            "resizable": True,
            "sortable": True,
            "flex": 1,
            "minWidth": 70,
        },
        "style": {"height": "100%"},
    }
    return AgGrid(
        columnDefs=[{"field": col} for col in chosen_columns],
        rowData=data_frame.to_dict("records"),
        **defaults,
    )

vm.AgGrid(

Custom figure
Design custom KPI cards or other figures with complex layouts and specialized visualizations. 
from vizro.models.types import capture
import vizro.models as vm
import pandas as pd
import dash_bootstrap_components as dbc
from dash import dcc, html

df = pd.DataFrame({"text": text * 2})

@capture("figure")
def multiple_cards(data_frame: pd.DataFrame, n_rows: Optional[int] = 1) -> html.Div:
    texts = data_frame.head(n_rows)["text"]
    return html.Div(
        [dbc.Card(dcc.Markdown(f"### Card #{i}\n{text}")) for i, text in enumerate(texts, 1)],
        className="multiple-cards-container",
    )

# Use in Figure component
vm.Figure(
    id="my-figure",
    figure=multiple_cards(data_frame=df)
)
